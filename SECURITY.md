# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in SickoMenu, please email security@example.com instead of using the issue tracker. This allows us to fix the issue before public disclosure.

Please provide:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Supported Versions

Security updates are provided for the following versions:

| Version | Supported          |
|---------|--------------     |
| 4.4     | :white_check_mark: |
| 4.3     | :white_check_mark: |
| < 4.3   | :x:                |

## Security Best Practices

1. Keep SickoMenu updated to the latest version
2. Review dependencies regularly
3. Use strong authentication
4. Follow the principle of least privilege

## Vulnerability Disclosure Timeline

1. Report received and confirmed
2. Fix development (typically within 7 days)
3. Security release
4. Public disclosure (after users have time to update)
