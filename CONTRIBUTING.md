# Contributing to SickoMenu

Thank you for your interest in contributing to SickoMenu! This document provides guidelines and instructions for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch (`git checkout -b feature/AmazingFeature`)
4. Make your changes
5. Commit your changes (`git commit -m 'Add some AmazingFeature'`)
6. Push to the branch (`git push origin feature/AmazingFeature`)
7. Open a Pull Request

## Code Guidelines

- Write clean, readable code
- Follow the existing code style and conventions
- Add comments for complex logic
- Test your changes before submitting

## Reporting Bugs

- Use GitHub Issues to report bugs
- Clearly describe the issue
- Include steps to reproduce
- Provide system information and environment details

## Feature Requests

- Use GitHub Issues for feature requests
- Describe the use case and benefits
- Discuss with maintainers before implementing

## Pull Request Process

1. Ensure your code follows project guidelines
2. Update documentation as needed
3. Link related issues in the PR description
4. Request review from maintainers
5. Address any requested changes

## Questions?

Feel free to open an issue or reach out to the community on Discord at discord.gg/sickos
